# Version Control Guidelines

- Semantic commit messages (feat/fix/docs/chore/refactor/test).
- Include attestation block in commit description:
```
attested_by: [Name]
verified_on: [Date]
node: [System / Canon Path]
```
- Branch policy:
  - `main`: stable, attested releases
  - `develop`: integration
  - `drafts`: pre-attestation work
- Issue labels: [companion], [ethics], [memory], [infra]
